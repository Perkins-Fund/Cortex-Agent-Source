# Code of Conduct

Cortex Agent Source is a community project. We’re here to build reliable endpoint agents and help people use them responsibly.

**Be respectful. Be constructive. Keep it safe.**

---

## Our Pledge
We pledge to make participation in this project a harassment-free experience for everyone, regardless of background or identity. We value professionalism, kindness, and constructive collaboration.

---

## Expected Behavior
Please:
- Be respectful and professional in issues, pull requests, and discussions
- Assume good intent; ask clarifying questions before escalating
- Provide constructive feedback and focus on ideas, not people
- Keep conversations on-topic and helpful
- Welcome newcomers and help others learn

---

## Unacceptable Behavior
The following is not tolerated:
- Harassment, hate speech, discrimination, or derogatory comments
- Personal attacks, insults, or bullying
- Threats of violence or encouraging harm
- Doxxing or sharing someone’s private information
- Spam, trolling, or intentionally disruptive behavior
- Posting secrets (API keys/tokens/passwords), private configs, or sensitive user/customer data

---

## Responsible Use (Security / Malware Context)
This repository supports malware analysis workflows. That does **not** mean we allow:
- Sharing live malware payloads or distribution links
- Guidance intended for wrongdoing or misuse
- Instructions meant to compromise systems or bypass protections

Keep content defensive, educational, and responsible.

If you believe you found a security vulnerability, **do not** post details publicly. See `SECURITY.md` for private reporting.

---

## Scope
This Code of Conduct applies to all project spaces including:
- GitHub Issues
- Pull Requests
- Discussions
- Code reviews
- Any community interactions related to this repository

---

## Enforcement
Maintainers may take action at their discretion to keep the community safe and productive, including:
- Removing or editing comments/content
- Issuing warnings
- Temporarily or permanently banning contributors
- Locking threads/issues/PRs

---

## Reporting
If you experience or witness unacceptable behavior, please report it privately.

Use GitHub’s reporting tools where appropriate, and/or contact the maintainers through Traceix support channels.

When reporting, include:
- A link to the relevant thread/content
- What happened and when
- Any context that may help with investigation
